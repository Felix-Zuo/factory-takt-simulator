# Security Policy

Do not submit private factory data, customer names, operator names, credentials, tokens, or internal network details in public issues.

If you find a security problem:

1. Do not publish a proof of exploit in a public issue.
2. Open a private advisory on GitHub if the repository enables it.
3. Include reproduction steps, affected version, and impact.

This project is a local simulation tool. It does not require a backend service by default.
