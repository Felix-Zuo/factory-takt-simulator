# Factory Takt Simulator 使用说明

## 1. 启动

开发预览：

```bash
npm install
npm run dev
```

桌面预览：

```bash
npm run desktop
```

打包 Windows 便携版：

```bash
npm run dist:win
```

## 2. 基本搭建

1. 在左侧模块库拖入 `SRC` 或 `FEED` 作为料源。
2. 拖入加工、检测、清洗、缓存、装配或包装模块。
3. 拖入 `END` 作为终点。
4. 从设备右侧输出口连接到下游左侧输入口。
5. 点击设备或连线，在右侧面板调整参数。

## 3. 设备参数

设备支持两种节拍模式：

- 详细模式：通过加工时间、修整、耗材更换、良率和稼动率计算综合节拍。
- 直接节拍模式：直接输入单件节拍，适合快速估算。

常用字段：

- 单次加工数量
- 加工一轮时间
- 前缓存容量
- 后缓存容量
- 当前缓存数量
- 设备数量
- 良率
- 稼动率

## 4. 连线参数

普通输送线：

- 运输时间
- 单次运输数量
- 在途容量
- 线体缓存

机械手：

- 触发批量
- 单次夹取数量
- 取料时间
- 移动时间
- 放料时间
- 返回时间
- 机械手组 ID

多条机械手路线使用同一个组 ID 时，画布会把它们显示成一条机械手总线。

## 5. 端口规则

点击端口可以设置简单路由规则：

- 允许通过的物料类型
- 轮询
- 堵塞跳过
- 分类输出

这些规则用于表达分流、合流、轮流上料和分类输送。

## 6. 保存与导入

顶部“保存 / 加载”可以打开方案库。支持：

- 保存当前方案
- 加载已有方案
- 删除方案
- 导出 JSON
- 导入 JSON
- 导出仿真报告
- 导出记录 CSV

## 7. 后台仿真

在“更多”菜单中打开后台仿真报告，可以选择：

- 按仿真时间运行
- 按目标产量运行

后台仿真不依赖画布动画，适合快速得到报告。

## 8. 智能体接入

应用启动后会暴露：

```ts
window.FactoryTaktAgent
```

可以用它读取当前方案、导入方案、控制仿真、运行后台仿真和获取报告。详细说明见：

```text
docs/AGENT_INTEGRATION.md
```
