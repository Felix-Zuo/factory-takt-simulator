# Factory Takt Simulator 使用教程

本教程面向第一次打开项目的使用者，说明如何从空白画布搭建一条可仿真的产线。

## 1. 启动项目

开发模式：

```bash
npm install
npm run dev
```

桌面模式：

```bash
npm run desktop
```

## 2. 添加工序

在左侧模块库中选择设备模块，拖到中间画布。常见模块包括：

- 料源 / 储料
- 加工设备
- 检测设备
- 清洗 / 甩干 / 缓存设备
- 合流 / 装配 / 包装终点

画布支持拖动、缩放、框选和网格对齐。

## 3. 连接工序

每个卡片左右两侧有输入口和输出口：

- 从输出口拖到另一个卡片的输入口，可以创建连线。
- 点击输出口后再点击输入口，也可以创建连线。
- 右键卡片、端口或连线，可以打开上下文菜单。

连线不是设备卡片，它表示两个工序之间的物流路径。

## 4. 设置设备参数

点击一个设备卡片，右侧参数面板会显示该设备的参数。

常用参数：

- 单次加工数量
- 加工一轮时间
- 前缓存容量 / 后缓存容量
- 当前缓存数量
- 设备数量
- 稼动率
- 良率
- 修整间隔和修整时间
- 耗材更换间隔和更换时间

设备可以使用两种节拍模式：

- **详细模式**：由加工、修整、耗材更换等参数自动计算综合节拍。
- **直接节拍模式**：直接输入单件综合节拍，适合快速估算。

## 5. 设置连线参数

点击连线，右侧会显示连线参数。

普通输送线：

- 运输时间
- 单次运输数量
- 在途容量
- 是否连续输送
- 是否允许缓存

机械手 / 上料手：

- 触发批量
- 单次夹取数量
- 取料时间
- 移动时间
- 放料时间
- 返回时间
- 等待上游达到批量
- 等待下游空间

机械手启动前会同时判断上游是否有足够物料、下游是否有足够空间。

## 6. 使用端口规则

端口可以设置简单的路由规则：

- 允许通过的物料类型
- 优先策略
- 堵塞时是否跳过
- 是否轮询多个端口

这些规则用于表达分流、合流、轮流上料、分类输送等现场常见逻辑。

## 7. 开始仿真

点击顶部“开始”后，系统会推进离散仿真：

- 设备从前缓存取料。
- 加工完成后尝试放入后缓存。
- 下游空间不足时进入堵料。
- 前缓存不足时进入待料。
- 达到修整或耗材更换条件时，设备会真实停机对应时间。
- 连线上的物料点会按运输时间沿路径移动。

## 8. 查看分析结果

右侧和底部面板会显示：

- 整线能力
- 瓶颈工序
- 工序累计产出
- 设备利用率
- 待料时间
- 堵料时间
- 优化建议

检测、缓存、料源和终点通常属于冗余承接环节，只有真正拖慢主线或造成堵料时才应作为主要瓶颈处理。

## 9. 保存、导入、导出

点击顶部“保存 / 加载”可以管理本地方案。更多菜单中可以：

- 导出方案 JSON
- 导入方案 JSON
- 导出仿真报告
- 导出记录 CSV
- 后台仿真生成报告

本地保存使用浏览器或 Electron 本地存储，不需要服务器。
